Crie um projeto web completo em Next.js (App Router) + TypeScript + Tailwind, baseado nas telas do site de tapeçaria (sofás, cabeceiras, arte em tecido, guias e consultoria).

OBJETIVO PRINCIPAL
Padronizar a aplicação para que TODAS as páginas usem EXATAMENTE o mesmo Header e Footer da Home, aplicados via layout global (app/layout.tsx). Não pode existir header/footer duplicado dentro das páginas.

ARQUITETURA OBRIGATÓRIA (feature-based)
A estrutura do código deve ser baseada em:
features/{route}/components
features/{route}/sections
features/{route}/data
features/{route}/types
features/{route}/utils
e apenas componentes realmente globais ficam em /components.

- Next.js (App Router)

- lucide-react (ícones)
- (Opcional) shadcn/ui apenas se necessário, mas prefira Tailwind puro

REGRAS DE LAYOUT (CRÍTICAS)
1) Header e Footer são globais e únicos, devem existir SOMENTE em:
   - /components/layout/Header.tsx
   - /components/layout/Footer.tsx
2) O app/layout.tsx deve ser o ÚNICO lugar onde Header/Footer aparecem.
3) Páginas em /app/* NÃO podem importar Header/Footer diretamente.
4) Botão flutuante do WhatsApp deve aparecer em TODAS as páginas via layout global:
   - /components/layout/WhatsAppFloatingButton.tsx
5) Header precisa ter:
   - Logo "LINEAR STORE"
   - links: Início, Sofás, Cabeceiras, Arte em Tecido, Processo, Guia de Design
   - botão "Fale com um especialista"
   - menu mobile hamburger funcional
6) Footer precisa ter:
   - coluna marca/descrição
   - coluna coleções (links)
   - coluna contato (endereço + telefone)
   - ícones sociais
7) Responsivo: layout mobile deve ficar semelhante ao screenshot (cards grandes, CTA visível, espaçamentos).

PÁGINAS E IMPLEMENTAÇÃO
- Home (/app/page.tsx): montar UI usando features/home/sections.
- Coleções (/sofas, /cabeceiras, /arte-em-tecido):
  reutilizar features/collections (mesmos componentes), alterando apenas o dataset (category).
- Processo (/processo): usar features/processo/sections.
- Guias (/guia-de-design): lista com cards de artigos usando features/guides.
- Guia detalhe (/guia-de-design/[slug]): página com hero do artigo + conteúdo + sidebar com tabela de referência + premium highlight + mais guias.
- Produto detalhe (/produto/[slug]): galeria, specs e CTA, com produtos relacionados.

DESIGN (seguir estilo das telas)
- Visual premium, tons neutros, botões arredondados
- Hero com imagem de fundo e gradiente escuro

- CTAs iguais ao conceito das telas ("Conhecer Coleção", "Agendar Consultoria", "Conversar no WhatsApp")
